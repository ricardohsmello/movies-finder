The Purpose of Movie Finder

Ever spent 30 minutes browsing only to give up on picking a movie? That’s why I built Movie Finder — to cut through indecision and offer suggestions based on how you feel, not just titles or actors.

Just type something like “a movie about pyramids, ancient Egypt, and a spaceship” or “the greatest World Cup champions of all time”, and let the app serve you intelligent, context-aware results.

Under the Hood

Frontend

The interface is a single static HTML page styled with CSS and a touch of JavaScript — designed to be simple, responsive, and fast.

Backend

The API is built with Quarkus, selected for its performance and rapid startup. It’s containerized via a custom Dockerfile and deployed on Render for hassle-free hosting.

MongoDB Atlas + Vector Search

The application connects to a MongoDB Atlas cluster that supports Vector Search for semantic retrieval. The index is defined with dotProduct similarity and 1536 dimensions: 

{
  "fields": [
    {
      "numDimensions": 1536,
      "path": "movie_embedded",
      "similarity": "dotProduct",
      "type": "vector"
    }
  ]
}

Each vector (movie_embedded) is precomputed using a combination of fields like title, plot, cast, and genres, enabling rich, semantically-aware search capabilities.
